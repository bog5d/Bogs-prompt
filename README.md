# Bogs-prompt

波总（Bogs）个人核心指令资产库 —— 数字思想副本与外挂大脑。

---

## 设计原则

| 原则 | 说明 |
|------|------|
| **SSOT** | 单点事实源。所有高频、沉淀性 Prompt 和思维模型只在此仓库进行版本迭代，抛弃一切传统笔记软件。 |
| **指针式提示词** | 将冗长逻辑封装在远端 `.md` 文件中；各端 AI 工具作为"无状态执行器"，通过 Raw URL 或剪贴板注入来加载上下文。 |
| **极度降阻** | 仓库设为 **Public** 是刻意的工程妥协，以换取移动端脚本免鉴权、零摩擦直接拉取。此处只存放处理逻辑与思维框架，不存放任何敏感业务数据。 |

---

## 调用链路

- **移动端（轻量会话）**：输入全局命令（如 `ksrj`），脚本自动拉取最新 `.md` 并注入剪贴板，手动喂给任意对话式 AI。
- **桌面端/开发端（重火力执行）**：将 `.md` 的 Raw URL 直接喂给 Claude Code / Cursor / Aider 等 Agent，让 Agent 顺藤摸瓜加载底层逻辑。

---

## 命名铁律

> 所有提示词文件必须严格遵守：`[拼音/英文极简缩写].md`
> 全小写，无空格，无特殊字符。

示例：`ksrj.md`（口述日记 Kou Shu Ri Ji）

---

## 版本迭代机制

采用**人机耦合**迭代：

- **波总**：在实际使用中发现旧版 Prompt 的缺陷，抛出原始想法和认知补丁。
- **AI（Claude Code 等）**：充当"编译器"，将碎片化指令结构化、更新对应 `.md` 文件，并执行底层的 `git commit` 与 `push`。
